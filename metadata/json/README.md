# NFT Metadata JSON

This directory stores generated metadata JSON files before upload to IPFS.

## Metadata Structure

Each JSON file follows the ERC-721 metadata standard:

```json
{
  "name": "CrashBytes NFT #1",
  "description": "A unique digital collectible from the CrashBytes collection",
  "image": "ipfs://QmImageHash...",
  "attributes": [
    {
      "trait_type": "Background",
      "value": "Blue"
    },
    {
      "trait_type": "Body",
      "value": "Robot"
    },
    {
      "trait_type": "Rarity",
      "value": "Legendary"
    }
  ]
}
```

## Required Fields

- **name**: Display name for the NFT
- **description**: Detailed description of the token
- **image**: IPFS URI pointing to the artwork

## Optional Fields

- **attributes**: Array of traits for marketplace filtering
- **external_url**: Link to external website
- **background_color**: Hex color for OpenSea background
- **animation_url**: For video/3D NFTs
- **youtube_url**: For video content

## Generation Process

Metadata files are automatically generated by the upload-metadata.js script:

1. Reads images from metadata/images/
2. Uploads images to IPFS
3. Creates metadata JSON with image CID
4. Saves JSON to this directory
5. Uploads JSON to IPFS
6. Returns metadata URI for minting

## Manual Creation

You can also manually create metadata files here and upload them separately:

```bash
node scripts/upload-metadata.js
```

## Marketplace Compatibility

This metadata format is compatible with:

- OpenSea
- Rarible
- LooksRare
- Foundation
- All ERC-721 compatible marketplaces
